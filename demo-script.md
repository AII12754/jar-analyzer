# jar-analyzer Web 工作台演示操作流程与口稿

> 基于 commit `5508f1a7` 实际代码编写。所有功能描述以代码为准。

---

## 演示前准备

1. 启动 jar-analyzer GUI：
   ```bash
   java -jar jar-analyzer-5.19-jar-with-dependencies.jar gui
   ```
2. 在 GUI 中导入演示样本 JAR（`test/innovation-demo/target/innovation-demo-0.0.1-SNAPSHOT.jar`）并完成数据库构建
3. 打开浏览器访问 `http://127.0.0.1:10032`，确认 Web 工作台页面加载成功

---

## 演示操作流程（约 8 分钟）

### 阶段一：运行状态概览（30 秒）

**页面**：概览（默认首页）

**操作**：
1. 页面自动加载，展示四张状态卡片：引擎状态、已加载 JAR、一键分析缓存、Server Auth
2. 点击"刷新状态"按钮
3. 指出引擎状态显示"Ready"，JAR 数量为 1

**演示要点**：
- 这是纯浏览器的 Web 工作台，不需要安装任何客户端
- 四大指标一眼看清当前分析环境

---

### 阶段二：方法检索（40 秒）

**页面**：方法检索

**操作**：
1. 点击左侧导航"方法检索"
2. 点击"Spring Controllers"按钮 → 展示所有 Spring Controller 类
3. 在"按字符串检索方法"输入框中输入 `exec`，点击"字符串检索" → 展示所有包含 `exec` 字符串的方法
4. 点击其中一条结果的"选中"按钮 → 该方法自动送入方法工作台，左侧"当前选中方法"卡片同步更新

**演示要点**：
- 搜索结果可以直接点击"选中"一键送入工作台，不需要手动复制粘贴类名
- 支持按类名搜索、按字符串（LDC 指令）搜索、加载 JAR 列表、加载 Spring Controllers

---

### 阶段三：方法工作台 — 调用关系与反编译（1 分 30 秒）

**页面**：方法工作台

**操作**：
1. 确认左侧"当前选中方法"卡片展示了刚选中的方法
2. 点击"调用者"按钮 → 查看谁调用了这个方法
3. 点击"被调用"按钮 → 查看这个方法调用了谁
4. 点击"CFR 反编译"按钮 → 展示完整方法级代码和类级代码（可折叠展开）
5. 点击"调用图"按钮 → 下方 iframe 渲染 D3.js 交互式方法调用图（节点可拖拽、缩放）
6. 点击"新窗口打开调用图" → 独立窗口展示完整交互图

**演示要点**：
- 方法工作台 = 一个方法的所有信息一站聚合：调用者/被调用/实现类/反编译/调用图
- 反编译是方法级精确提取（不是给你整个类文件），带代码高亮
- 调用图是 D3.js 交互式力导向图，不是静态图片

---

### 阶段四：DFS 分析与污点图（1 分 30 秒）

**页面**：DFS 分析

**操作**：
1. 点击左侧导航"DFS 分析"
2. 指出页面顶部有内置预设快捷按钮：Runtime.exec、ProcessBuilder.start、JNDI lookup 等
3. 点击"Runtime.exec"预设 → Sink 参数自动填充（`java/lang/Runtime` / `exec`）
4. 点击"执行 DFS" → 展示 DFS 搜索结果列表，列出从入口到 Runtime.exec 的调用链
5. DFS 执行后自动生成污点图，下方 iframe 展示交互式 D3.js 图
6. 点击"新窗口打开污点图"

**演示要点**：
- 8 个内置预设一键填充（命令执行、JNDI、反序列化、脚本执行、SSRF、文件写入、反射）
- DFS 反向搜索（从 Sink 向上追溯）自动找到 Web 入口 → 危险调用的完整链路
- 污点图可视化展示传播路径

---

### 阶段五：安全巡航 — 亮点功能（2 分钟）

**页面**：安全巡航

**操作**：
1. 点击左侧导航"安全巡航"
2. 页面自动调用 `/api/security_overview`，展示：
   - **风险摘要卡片**（4 张）：Web 入口资产数、Spring 路由数、命中风险类别数、风险得分
   - **优先排查方向**：按严重程度排序的命中排行，带比例条可视化
   - **入口资产概览**：Spring Controllers / Servlets / Filters / Listeners 数量和各取前 5 个类名
   - **内置漏洞排查**：7 类自动排查结果（命令执行、JNDI、反序列化、脚本执行、SSRF、文件写入、反射调用），每类展示前 5 条命中，带"送入工作台"和"调用图"按钮
   - **DFS 快速预设**：8 个预设卡片，可一键套用到 DFS 面板
   - **Spring 入口映射**：HTTP 方法 + 路径 + 类名 + 方法名
3. 在命令执行排查结果中，点击某条命中的"送入工作台"按钮 → 自动跳转到方法工作台
4. 回到安全巡航，点击某条命中的"调用图"按钮 → 自动跳转到方法工作台并生成调用图
5. 在优先排查方向中，点击某条旁的"DFS 预设"按钮 → 自动跳转到 DFS 面板并填充参数

**演示要点**：
- 安全巡航是"总控中心"：一次性聚合 Web 入口资产 + 危险 Sink 调用者 + 路径映射
- 风险得分是动态计算的（入口数×2 + 映射数 + 命中数×严重权重），不是拍脑袋的数字
- 每条发现都可以"送入工作台"继续深挖 → "调用图"可视化 → "DFS 预设"追调用链
- 从巡航到深挖是一键跳转的闭环，不用离开浏览器

---

### 阶段六：LLM 审计报告（1 分 30 秒）

**页面**：LLM 报告

**操作**：
1. 点击左侧导航"LLM 报告"
2. 填写 LLM 配置：
   - API Endpoint（默认 `https://api.openai.com/v1/chat/completions`，也支持本地 Ollama/DeepSeek 等）
   - API Key
   - 模型名称（默认 `gpt-4o-mini`）
3. 勾选"先执行整项目自动分析"（或直接"仅基于缓存生成"）
4. 点击"一键分析并生成报告"
5. 右侧日志区实时展示三个阶段进度：
   - 阶段 1/3：过程间数据流分析（DFA）
   - 阶段 2/3：反序列化利用链自动挖掘
   - 阶段 3/3：调用 LLM 生成最终审计报告
6. 生成完成后，点击"打开独立报告页" → 新标签页展示精美排版的安全审计报告（含风险摘要卡片、目录导航、漏洞台账）
7. 独立报告页完全自包含（CSS 内联），可直接另存为 HTML 交付

**演示要点**：
- 兼容任何 OpenAI API 格式的服务（ChatGPT、DeepSeek、Ollama、本地 vLLM）
- 一键完成"分析 + 报告生成"，也可以分步：先分析（缓存结果），再多次生成报告
- LLM 只负责"写作"（将结构化分析结果转化为专业报告），安全分析本身是确定性的 Java 算法
- 报告是完整 HTML 页面，带响应式排版、目录导航、风险摘要卡片

---

### 阶段七：浏览器构建（快进展示，30 秒）

**页面**：浏览器构建

**操作**：
1. 点击左侧导航"浏览器构建"
2. 指出可以通过两种方式导入：
   - 上传归档文件（拖拽或选择）
   - 填写服务器本机路径（更快，不需要上传）
3. 三个构建选项：quick_mode、fix_class、clean_before_build
4. 说明构建过程有实时进度条和日志输出，每 1.8 秒自动轮询状态

**演示要点**：
- 完全在浏览器中完成 JAR/WAR 导入和数据库构建，不需要操作 GUI 客户端
- 构建进度实时反馈，完成后自动刷新引擎状态

## 演示口稿

---

### 开场

大家好，我今天演示的是 jar-analyzer 的**浏览器工作台**。

jar-analyzer 是一个开源的 Java JAR 包安全分析工具，我们为它构建了一套完整的 Web 前端，让安全研究人员可以在浏览器里完成从 JAR 导入、方法检索、调用链追踪、漏洞排查，到最终 LLM 审计报告生成的全流程。

整个前端是一个纯静态的单页应用，通过 jar-analyzer 内置的 HTTP API 与后端通信，共 7 个功能页面，约 1600 行 JavaScript。

---

### 01 运行状态概览

我们先看概览页。

页面加载后自动请求服务端状态，四张卡片告诉我们：引擎已经是 Ready 状态，加载了 1 个 JAR 包，一键分析缓存还未生成，Server Auth 是关闭的。

这个页面的意义在于：打开浏览器就能确认分析环境是否就绪。

---

### 02 方法检索

接下来进入方法检索页。

这里支持三种检索方式：按类名搜索、按字符串常量搜索、以及一键加载 Spring Controllers。

我点击"Spring Controllers"，立刻列出所有 Controller 类。

再输入 `exec` 做字符串检索——这是搜索字节码中 LDC 指令加载的字符串常量——结果列出了所有包含 `exec` 的方法。

关键操作：点"选中"，这个方法就直接送入了方法工作台，左侧卡片同步更新。不需要手动复制类名方法名描述符。

---

### 03 方法工作台

现在进入方法工作台。

这是我选中的方法，我可以一键查看：谁调用了它、它调用了谁、它的实现类有哪些、以及 CFR 和 Fernflower 两种反编译引擎的结果。

点击 CFR 反编译——展示的是方法级精确提取的代码，不是整个类文件。下面还可以展开完整类代码。

再点击调用图——下方立刻渲染出 D3.js 交互式力导向图，节点可以拖拽，滚轮缩放。这是给安全研究员看的：一眼就能看出这个方法的上下游调用关系。

---

### 04 DFS 分析与污点图

现在进入 DFS 分析页。

这个页面的核心能力是**从危险 Sink 反向追踪调用链**。

页面顶部有 8 个内置预设按钮——Runtime.exec、ProcessBuilder.start、JNDI lookup、反序列化 readObject、脚本执行 eval、SSRF、文件写入、反射调用。

我点一下 Runtime.exec，Sink 参数就自动填好了。不需要手写 JVM 内部类名。

点击执行 DFS——搜索从任意 Web 入口出发，到达 Runtime.exec 的调用链。同时自动生成污点图，用 D3.js 可视化展示污点传播路径。

---

### 05 安全巡航 — 核心亮点

这是整个工作台最核心的页面：安全巡航。

它的设计理念是：**先做全局筛查，再定向深挖**。

页面加载后自动调用 security_overview API，一次请求拿到所有信息：

**上面四张卡片**是风险摘要——Web 入口资产数量、Spring 路由数、命中几类风险、以及一个动态计算的风险得分。

**优先排查方向**按严重程度排序——严重和高危的排在前面，带比例条可视化，一眼看出哪些方向最需要关注。

**入口资产概览**列出 Controllers、Servlets、Filters、Listeners 的统计和类名。

**内置漏洞排查**是 7 类自动 Hunt 的结果——命令执行、JNDI、反序列化、脚本执行、SSRF、文件写入、反射调用。每一类都精确到"哪个类的哪个方法调用了危险 Sink"。

关键来了——每条命中都有两个操作按钮：**"送入工作台"**和**"调用图"**。

点"送入工作台"，自动跳转到方法工作台，类名方法名全部填好，接着就能查调用者、看反编译。

点"调用图"，直接生成这个方法的交互式调用图。

下面还有 **Spring 入口映射**——每个 HTTP 端点的路径、HTTP 方法、对应哪个类的哪个方法，同样可以送入工作台或直接生成调用图。

最下面是 **DFS 快速预设**——8 张卡片，点任意一张直接跳转到 DFS 面板并填充好参数。

整个安全巡航的设计闭环是：**发现 → 送入工作台深挖 → 生成调用图验证 → DFS 追完整链路**，全部在浏览器中一键跳转完成。

---

### 06 LLM 审计报告

最后是 LLM 审计报告页。

这个页面集成了大模型能力来生成专业的安全审计报告。

配置很简单——填 Endpoint、API Key、模型名。兼容任何 OpenAI API 格式的服务，你可以用 ChatGPT，也可以用 DeepSeek、Ollama 本地模型，保护代码隐私。

点击"一键分析并生成报告"，后台依次执行三个阶段：过程间 DFA 数据流分析、反序列化利用链自动挖掘、最后把所有结构化发现发送给 LLM，生成专业审计报告。

注意：LLM 只负责"写作"——把确定性的分析结果转化为人类可读的报告。安全分析本身是 Java 算法完成的，不是让 AI 去猜哪里有漏洞。

生成完成后，点击"打开独立报告页"——这是一份完整排版的安全审计报告，带风险摘要卡片、侧边栏目录导航、漏洞台账。整个 HTML 文件是自包含的，CSS 全部内联，可以直接保存交付。

---

### 07 浏览器构建

最后快速展示一下构建能力。

在浏览器构建页，你可以直接上传 JAR/WAR 文件，或者填写服务器本机路径，完全在浏览器中完成分析索引的构建。

构建过程有实时进度条和日志输出。

---

### 总结

jar-analyzer 浏览器工作台的核心价值在于：

1. **零客户端依赖**——打开浏览器就能完成全部安全分析工作
2. **安全巡航 + 定向深挖闭环**——从全局筛查到单方法深挖，一键跳转
3. **DFS + 污点图可视化**——调用链不再是一堆文字，而是交互式图形
4. **LLM 智能报告**——兼容任意模型，确定分析 + AI 写作，报告可直接交付
5. **完全开源**——前端约 1600 行 JS，后端 API 全部开放

谢谢大家。
